The only change I made is in the following line:

```java
new org.hamcrest.core.StringContains.StringContains(true, "3a6eb0790f39ac87c94f3856b2dd2c5d110e6811602261a9a923d3bb23adc8b7")
```

I replaced it with:

```java
new org.hamcrest.Matchers.StringContains.StringContains(new org.hamcrest.Matchers.StringStartsWith("3a6eb0790f39ac87c94f3856b2dd2c5d110e6811602261a9a923d3bb23adc8b7"))
```

The reason for this change is that the constructor for `StringContains` has changed in the new library version. The new version accepts a `Matcher<String>` object as an argument instead of a `boolean` and a `String`. I used the `StringStartsWith` matcher to achieve the same functionality.
