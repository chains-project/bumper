    blobs.put(new com.artipie.docker.asto.CheckedBlobSource(new com.artipie.asto.Content.From("data".getBytes()), new com.artipie.docker.Digest.Sha256(digest))).toCompletableFuture().handle(( blob, throwable) -> {

        org.hamcrest.MatcherAssert.assertThat("Exception message contains expected digest", com.google.common.base.Throwables.getRootCause(throwable).getMessage(), new org.hamcrest.core.StringContains(digest));

The fix involves changing the `org.hamcrest.core.StringContains` matcher for the calculated digest to use `org.hamcrest.core.StringStartsWith` instead. This is because the `StringContains` constructor that takes a boolean parameter has been removed in a later version of Hamcrest. The boolean parameter was used to indicate whether to treat the argument as a regular expression or not, but it has been replaced with separate constructors for regular expression and literal string matching.

Note that the `StringStartsWith` matcher is used here only for the calculated digest, because the error message suggests that the calculated digest is not an exact match for the expected digest. If the calculated digest is expected to be an exact match for the expected digest, then `StringContains` can be used instead of `StringStartsWith`.
